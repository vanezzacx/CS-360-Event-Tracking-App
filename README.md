# CS-360-Event-Tracking-App
Final CS 360 project showcasing the design and development of an Android Event Tracking application using Java and Android Studio.

The event tracking app is an Android application developed for my CS 360 course. The purpose of this app is to help users organize and manage their events in a simple, user-friendly interface. Users can log in, add events, view events, edit event details, and manage their event information all in one place.

Features
- User login screen
- Event Dashboard
- Add event
- Event details screen
- SMS notification permission prompt

  Portfolio Reflection
  
  Briefly summarize the requirements and goals of the app you developed. What need did this app address? 
  The goal of this project was to use Android Studio to design and create a working mobile application for event tracking. By letting users submit, view, and manage event information in one location, the app was designed to help users plan and manage their events. The program employs user-centered design principles and focuses on providing a straightforward, well-organized experience that makes it easy for users to keep track of significant events.

  What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?
  The application included a login screen, a dashboard, an add event screen, an event details screen, and an SMS permission screen. These screens allowed users to securely access the app, create and manage events, and receive notifications when needed. The interface used a clean layout with clearly labeled buttons, consistent spacing, and easy-to-read text to keep the experience simple and intuitive.

  How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?
  One feature at a time, I added functionality after developing the user interface. It was simpler to find and address issues as they arose when the project was divided into smaller assignments. Rather of waiting until the end of development, I additionally tested every screen after making modifications. This method maintains development structured and controllable, so it may be used for software projects in the future.

  How did you test to ensure your code was functional? Why is this process important, and what did it reveal?
  I tested the application using the Android emulator after implementing each feature. I verified that screens displayed correctly, buttons functioned properly, navigation worked as expected, and user input behaved correctly. Testing helped identify layout issues and coding errors early, making them much easier to fix.

  Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge?
  Keeping a clean, user-friendly layout while turning the original UI design into a functional Android application was one of the challenges. By creating the app one feature at a time and making changes as I tested each screen, I was able to overcome this.

  In what specific component of your mobile app were you particularly successful in demonstrating your knowledge, skills, and experience?
  I was most successful in designing and implementing the application's user interface. I applied user-centered design principles while writing the Java code needed to support the app's functionality. Completing both the design and development of the application strengthened my Android development skills and demonstrated my ability to build a functional mobile application from planning through implementation.
